# Contributing Guidelines

Thank you for considering contributing to Awesome Mobula!

## Guidelines

- **Quality over quantity**: Only submit projects that are actively maintained and provide value
- **Must use Mobula**: All listed projects must integrate Mobula API
- **Working projects only**: Ensure the project is functional and accessible
- **Proper formatting**: Follow the existing format (name + brief description)
- **Appropriate category**: Place your submission in the correct section

## How to Contribute

1. Fork this repository
2. Create a new branch (`git checkout -b add-project-name`)
3. Add your project to the appropriate section in alphabetical order
4. Format: `- [Project Name](link) - Brief description`
5. Commit your changes (`git commit -m 'Add Project Name'`)
6. Push to the branch (`git push origin add-project-name`)
7. Open a Pull Request

## What to Include

- **Project name**: Clear and concise
- **Link**: Direct link to project (website, GitHub, etc.)
- **Description**: One-line description of what the project does
- **Category**: Place in the most relevant section

## What Not to Include

- Broken or unmaintained projects
- Projects without Mobula API integration
- Promotional content or spam
- Duplicate entries

## Questions?

If you have questions, open an issue or reach out on [Telegram](https://t.me/mobuladevelopers).

Thank you for helping make this list awesome!
